# AI Incident Response Playbook

**Status:** In progress — target completion Week 5 of the 12-week training plan.

An operational IR playbook for AI/LLM incidents covering at minimum: prompt injection leading to data exfiltration via RAG, training-data poisoning, hallucination-driven downstream harm, credential leakage through model outputs, and jailbreak-as-a-service abuse.

See [`../frameworks-cross-reference.md`](../frameworks-cross-reference.md) for the framework mapping that anchors the detection signals referenced in each scenario.
